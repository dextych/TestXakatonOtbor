# XakatonKubok - Тестовый фреймворк для игровых комнат
Набор автоматизированных тестов для проверки API игровых комнат, баланса пользователей и механики игры с бустами и ботами.

## Установка

```bash```
*npm install*

## Скрипты запуска

| Скрипт | Описание |
|---|---|
| `npm run user-generation` | Параллельная генерация тестовых пользователей |
| `npm run get-token` | Параллельное получение JWT токенов для созданных пользователей |
| `npm run join-room` | Тест входа игроков в существующую комнату |
| `npm run create-and-join` | **Основной тест:** полный цикл игры (создание → вход → боты → раунды → баланс) |
| `npm run create-scheduled-room` | Создание комнаты по расписанию (с повтором) |
| `npm run check-repeat` | Проверка создания повторной комнаты и игрового процесса в ней |
| `npm run test:balance` | Тестирование операций с балансом (публичный API) |

## Описание тестов

### 1. `create-and-join` — Полный игровой цикл

**Файл:** `mainTest/create-and-join-lobbies.spec.js`

Проверяет **end-to-end** сценарий:

- Создание комнаты с заданной конфигурацией
    
- Вход двух реальных игроков по WebSocket
    
- Проверка начального баланса
    
- Ожидание заполнения комнаты ботами
    
- **Раунд 1:** выбор 3 бочек, покупка и применение буста
    
- **Раунд 2:** выбор бочек для прошедших игроков
    
- Проверка финального баланса (изменения `available` и `reserved`)
    

---

### 2. `create-scheduled-room` — Комнаты по расписанию

**Файл:** `createScheduledRoom/create-scheduled-rooms.spec.js`

Создаёт комнату с параметрами:

- `scheduledStartAt` — время первого запуска
    
- `repeatInterval` — интервал повтора (`EVERY_30_MIN`, `EVERY_HOUR`)
    
- Оценка конфигурации через `/admin/evaluate`
    
- Ожидание активации комнаты (переход `SCHEDULED` → `WAITING`)
    

---

### 3. `check-repeat` — Проверка повторных комнат

**Файл:** `createScheduledRoom/check-repeated-room.spec.js`

Запускается **через 30+ минут** после создания комнаты с повтором:

- Находит все комнаты с одинаковой конфигурацией (исключая `SCHEDULED`)
    
- Входит в найденные комнаты игроками
    
- Проверяет полный игровой процесс в повторной комнате
    
- Убеждается, что комната работает так же, как и оригинальная
    

---

### 4. `test:balance` — Операции с балансом

**Файл:** `testBalance/balance-operations.spec.js`

Проверяет публичный API баланса:

- Получение баланса текущего пользователя (`/api/v1/users/me/balance`)
    
- Проверка структуры ответа (`available`, `reserved`)
    
- _Internal API не тестируется (доступно только внутри кластера)_
    

---

### 5. `user-generation` — Генерация пользователей

**Файл:** `userGeneration/user-generation.js`

Параллельное создание тестовых пользователей через API регистрации.
Количество и все остальные данные пользователей задаются в конфиге.
**Файл:** `config\testConfig.js` 

---

### 6. `get-token` — Получение токенов

**Файл:** `getTokenUser/get-token.js`

Параллельная авторизация созданных пользователей и сохранение JWT токенов в `tokens.json`.
Данные для токенов задаются в конфиге пользователя.
**Файл:** `config\testConfig.js` 


## Конфигурация (`config/testConfig.js`)

Все параметры тестов вынесены в единый конфигурационный файл.

### Сетевые настройки

| Параметр | Значение | Описание |
|---|---|---|
| `BASE_URL` | `http://92.51.23.102:8080` | Core-сервис (пользователи, балансы, WebSocket) |
| `GAME_URL` | `http://92.51.23.102:8081` | Game-сервис (комнаты, раунды, боты) |

### Авторизация

| Параметр | Описание |
|---|---|
| `ADMIN_TOKEN` | JWT токен администратора для создания комнат и доступа к internal API |
| `TOKENS` | Массив токенов тестовых пользователей (заполняется автоматически скриптом `get-token`) |

### Генерация пользователей (`USER_GENERATION`)

| Параметр | Пример | Описание |
|---|---|---|
| `count` | `60` | Количество создаваемых пользователей |
| `phonePrefix` | `'+7912383'` | Префикс номера телефона |
| `usernamePrefix` | `'dextytest'` | Префикс имени пользователя |
| `password` | `'123455cc'` | Общий пароль для всех тестовых пользователей |
| `phonePadLength` | `4` | Длина числового суффикса (0001, 0002...) |

### Конфигурация основной комнаты (`ROOM_CONFIG`)

| Параметр | Тип | Пример | Описание |
|---|---|---|---|
| `maxPlayers` | `int` | `5` | Максимальное количество игроков |
| `entryFeeAmount` | `number` | `100` | Входная плата в бонусных баллах |
| `winnerPayoutPercentage` | `number` | `80` | Процент приза победителю |
| `boostCostAmount` | `number` | `20` | Стоимость буста |
| `boostEnabled` | `boolean` | `true` | Доступен ли буст |
| `maxBarrelSelection` | `int` | `3` | Количество выбираемых бочек |

### Конфигурация комнаты по расписанию (`SCHEDULED_ROOM_CONFIG`)

Используется в тесте `create-scheduled-room`.

| Параметр | Пример | Описание |
|---|---|---|
| `name` | `'Комната с повтором каждые 30 мин'` | Название для логирования |
| `config.maxPlayers` | `7` | Максимум игроков |
| `config.entryFeeAmount` | `33` | Входная плата |
| `config.winnerPayoutPercentage` | `33` | Процент победителю |
| `config.boostCostAmount` | `20` | Стоимость буста |
| `config.boostEnabled` | `true` | Доступность буста |
| `config.maxBarrelSelection` | `3` | Количество бочек |
| `config.repeatInterval` | `'EVERY_30_MIN'` | Интервал повтора |
| `config.confirmWarnings` | `true` | Подтверждение предупреждений |

### Конфигурация поиска повторных комнат (`REPEAT_ROOM_SEARCH_CONFIG`)

Используется в тесте `check-repeat` для идентификации комнат, созданных по расписанию.

| Параметр | Пример | Описание |
|---|---|---|
| `maxPlayers` | `7` | Максимум игроков (должен совпадать с `SCHEDULED_ROOM_CONFIG`) |
| `entryFeeAmount` | `33` | Входная плата |
| `winnerPayoutPercentage` | `33` | Процент победителю |
| `boostCostAmount` | `20` | Стоимость буста |
| `repeatInterval` | `'EVERY_30_MIN'` | Интервал повтора |

### Прочие настройки

| Параметр | Пример | Описание |
|---|---|---|
| `LOBBIES_COUNT` | `1` | Количество комнат для создания в основном тесте |
| `PLAYERS_PER_LOBBY` | `4` | Количество пользователей в одной комнате |
